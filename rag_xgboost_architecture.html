System architecture: RAG feature extraction feeding XGBoost crime rate predictor

PHASE 1 — OFFLINE: BUILD KNOWLEDGE BASE
NLP pipeline — ingest & index NCRB PDF
01
PDF parsing
Extract text + table structure using pdfplumber. Separate prose sections from tabular rows. Tag each chunk with chapter, table ID, and page number.
pdfplumber
→
02
Chunking strategy
Use table-aware chunking — each row becomes a chunk tagged with state/city + metric type. Prose chunks are sentence-split with 50-token overlap.
LangChain splitter
→
03
Embedding
Encode all chunks with a domain-appropriate model. Each chunk → dense vector stored with metadata (state, year, table, metric).
sentence-transformers
→
04
Vector store
Store vectors + metadata in FAISS (local) or ChromaDB. Build a separate structured index for table rows to enable exact numeric lookup.
FAISS / Chroma
↓
PHASE 2 — OFFLINE: EXTRACT & STRUCTURE FEATURES
Feature engineering via structured RAG queries
05
Template queries
For each of 36 states, run a fixed set of RAG queries: "What is {state}'s IPC crime rate?", "What is {state}'s conviction rate for murder?", etc. These are not user queries — they are automated extraction calls.
36 states × 12 features
→
06
Number extraction NLP
Pass RAG response through a regex + NER layer to extract the numeric value. Handle units (per lakh, %, cases). Validate range — flag if value is implausible.
spaCy NER + regex
→
07
Feature matrix
Assemble a 36 × 12 DataFrame — one row per state, one column per extracted feature. Missing values flagged for imputation. This is the ML training input.
pandas DataFrame

PHASE 3 — OFFLINE: TRAIN XGBOOST
ML training — XGBoost regressor
08
Target variable
Primary target: overall IPC crime rate per lakh population (Table 1A.1). Secondary target: % YoY change 2021→2022. Train separate models per target.
regression target
→
09
Train / val split
With only 36 states, use leave-one-out CV or 5-fold CV. Augment with 2020 and 2021 data from prior NCRB reports to get 108 rows (36 × 3 years) — enough for robust training.
LOO-CV or k-fold
→
10
XGBoost + SHAP
Train XGBoost regressor with hyperparameter tuning via GridSearchCV. Run SHAP to get feature importance — this becomes a key result: which judicial/crime features most predict overall crime rate.
xgboost + shap
↓
PHASE 4 — ONLINE: INFERENCE PIPELINE
Live query → prediction
11
User query
"What will Rajasthan's crime rate trend look like given its current conviction rate?" Natural language input — no structured form required.
→
12
RAG retrieval
Query hits the vector store, retrieves the top-k chunks for Rajasthan across relevant tables. LLM generates a grounded answer with citations.
NLP output
→
13
Feature lookup
Identify the state from the query. Pull Rajasthan's feature row from the pre-built matrix. Feed into trained XGBoost model.
ML input
→
14
Combined output
Return RAG answer + predicted crime rate + SHAP explanation of which features drove the prediction. One unified response to the user's question.
unified response
